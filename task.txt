ТЗ на разработку проекта "Temp-Tracker"
Цель: Создать легковесный веб-интерфейс для быстрого ввода данных о температуре тела в Google Таблицу через смартфон.
1. Стек технологий
Frontend: Одностраничное приложение (HTML/CSS/JS), без тяжелых библиотек.
Backend: Google Apps Script (GAS) в режиме Web App.
База данных: Google Sheets.
2. Функциональные требования
Ввод данных:
Центральный элемент — крупные цифры текущего значения (по умолчанию 36.6).
Кнопки-стрелки "Вверх" и "Вниз" с шагом 0.1 для изменения значения.
Кнопка "Отправить данные" (большая, заметная).
Отображение истории:
Ниже области ввода — список последних 5-10 записей (Дата | Время | Температура).
Интеграция:
Отправка данных (температура и текущий timestamp) методом POST или GET на URL Google Apps Script.
Автоматическая запись данных в первую свободную строку Google Таблицы.
3. Требования к интерфейсу (UI/UX)
Mobile First: Дизайн под экран смартфона, отсутствие горизонтальной прокрутки.
Крупные элементы: Кнопки должны быть удобны для нажатия пальцем (минимум 44x44 px).
Запрет зума: Использовать метатег viewport с запретом масштабирования при случайном двойном тапе.
Обратная связь: После нажатия "Отправить" показывать четкий индикатор успеха (например, зеленая галочка или текст "Данные сохранены").
4. Структура данных (Google Sheets)
Колонка А: Дата и время (Timestamp).
Колонка B: Значение температуры.
5. Инструкции для ИИ (Prompts для реализации)
Шаг 1: Сгенерируй код для Google Apps Script (функция doPost), который принимает JSON с температурой и записывает её в активный лист.
Шаг 2: Создай адаптивный HTML/CSS файл с использованием Flexbox. Стилизуй его в духе минимализма с крупными шрифтами.
Шаг 3: Напиши JavaScript для обработки кликов по стрелкам, обновления состояния на экране и выполнения fetch запроса к GAS API.
